# A.3 강제 조건부 비교

이번 절에서는 조건에 맞는지 아닌지를 판단하려면 강제 변환이 선행되어야 하는 조건부 표현식에 대해 이야기 해보겠다.

if와 ?, :가 연달아 나오는 삼항 조건부 연산자 그리고 while과 for 반복문의 조건절은 암시적으로 값을 비교한다. 비교에는 타입이 같은지도 확인하는 엄격 비교가 있고, 강제로 타입을 전환해 비교하는 경우가 있다. 그렇다면 강제 조건부 비교는 어떤 비교 방식을 따르는 것일까? 답은 둘 둘다이다.

``` js
var x = 1;

if (x) {
    // 코드가 실행된다.
}

while (x) {
    // 코드가 딱 한번 실행된다.
    x = false;
}
```

아마도 여러분은 위 예시를 봤을 때, x 관련 조건부 표현식을 아래처럼 치환해서 생각했을 것이다.

``` js
var x = 1;

if (x == true) {
    // 코드가 실행된다.
}

while (x == true) {
    // 코드가 딱 한번 실행됩니다.
    x = false;
}
```

x가 1인 경우 특정한 사례에서는 이런 멘탈 모델이 잘 작동한다. 하지만 이 방법은 모든 경우에 사용할 수는 없다. 예시를 살펴보자

``` js
var x = "안녕하세요";

if (x) {
    // 코드가 실행된다.
}

if (x == true) {
    // 코드가 실행되지 않는다.
}
```
원하는대로 작동하지 않는다. 도대체 if 조건문은 실제로 어떤 일을 하기에 이런 결과가 나타난걸까? 이제 앞으로 우리 머릿속에 심어야 할 멘탈 모델은 다음과 같다.

``` js
var x = "안녕하세요";

if (Boolean(x) == true) {
    // 코드가 실행됩니다.
}

// 이렇게 치환해도 괜찮습니다.
if (Boolean(x) === true) {
    // 코드가 실행됩니다.
}
```

함수 Boolean()은 항상 불리언 타입의 값을 반환하므로 ==를 쓰든 ===를 쓰든 상관없이 결과는 같다. 중요한 것은 비교하기 전에 강제 변환이 일어나고, x는 타입에 상관없이 불리언 값이 된다는 점이다.
