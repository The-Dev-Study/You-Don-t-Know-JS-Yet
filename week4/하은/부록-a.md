## 값 vs 참조

값을 할당하거나 전달할 때 **값 그 자체**인지 **값의 참조**인지 개발자가 선택할 수 있도록 하는 프로그래밍 언어가 많지만

> JS에서는 **오로지 값의 타입으로 결정**된다.

### 원시 타입은 값의 복사

**원시 타입**: 값을 할당하거나 전달할 때 **값이 복사**된다.

```js
var myName = '카일';
var yourName = myName;
```

-   변수 `yourName` 에는 변수 `myName`에 할당된 값인 '카일'을 복사한 **독립된 문자열**이 저장된다.
-   두 변수에 저장된 값이 독립적이므로 두 변수가 서로 영향을 주지 못한다.

## 참조는 값의 참조 복사

참조를 복사하면 여러 변수가 하나의 값을 가리키게 되므로, 공유하는 값을 수정하면 이 값을 참조하는 모든 변수가 영향을 받는다.

> JS에서 객체 값만 참조로 처리된다.

```js
// myAddress에 객체를 할당 -> 참조가 저장되어 있다!
var myAddress = {
    street: '종로구 청와대로 1',
    city: '서울 특별시',
    country: '대한민국',
};

var yourAddress = myAddress; // 참조의 복사 -> 동일한 객체를 가리킴
myAddress.street = '양천구 중앙로';

console.log(yourAddress.street); // 양천구 중앙로
```
