# 3.4.1 객체 연결 장치
객체 프로토타입 연결 장치를 직접 정의하고 싶을 때는 Object.crate()를 사용해 객체를 만들면 된다.

``` js
var homework = {
    topic: "JS"
}

var otherHomework = Object.create(homework);
otherHomework.topic; // "JS"
```
Object.create()의 첫번째 인수에는 새롭게 생성할 객체를 어떤 객체와 연결할지 명시한다. 이렇게 해서 메서드를 호출하면 첫 번째 인수로 넘긴 객체와 연결된 새로운 객체가 반환된다.

프로토타입 체인을 통한 위임은 프로퍼티에 접근해 값을 찾을 때만 적용된다.
객체에 새로운 프로퍼티를 추가할 때는 해당 객체가 어떤 객체와 연결되어 있는지와는 상관없이 바로 그 객체에 프로퍼티가 추가된다.

> Object.create(null)을 호출하면 어떤 객체에도 연결되어 있지 않은 순수 독립 객체가 만들어진다. 이런 객체가 필요한 경우가 있으니 알아두자

앞선 예시와 이어지는 코드를 살펴보자
``` js
homework.topic;
// JS

otherHomework.topic;
otherHomework.topic = "수학";
otherHomework.topic;
// 수학

homework.topic;
// JS <- 수학이 아님
```

topic에 값을 할당하면 otherHomework 객체에 바로 해당 이름을 가진 프로퍼티가 만들어진다. 이때 homework 객체의 topic에는 영향이 하나도 없다.

> ES6에서 class 문법이 추가되기 전에는 프로토타입 연결장치를 사용해 겍체를 만들려면 프로토타입 클래스라는 복잡한 패턴을 사용했다.
