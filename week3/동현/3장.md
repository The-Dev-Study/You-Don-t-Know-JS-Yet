### 3.1 이터레이션

연속적이지 않은 자료구조를 순회하기 위한 컨테이너 구현 방식인 이터레이터에 대한 설명

### 3.1.1 이터레이터 소비하기

for of 문, 구조분해할당으로 간단하게 사용 가능

### 3.1.2 이터러블

array map set 같은 js내장 자료구조는 모두 이터러블하다

다만 메모리 인접 배열인 array와 달리(v8엔진으로 메모리 인접배열이 아닐수도 있다. 유니온타입 배열 insert시 구조가 강등되는 방식으로 최적화 되어 있음) map, set에 대한 내부 구조를 잘 이해하는게 중요하다

### 3.2 클로저

함수 내부에 선언되어 스택메모리에 들어가는 지역변수를 함수에서 리턴해 함수 외부에서 참조가 걸려 GC에 회수되지 않는 패턴을 일컫는다

하지만 참조타입이 아닌 object 제외 나머지 원시타입들 또한 클로저로 안전하다

렉시컬 환경 자체(걍 object라 생각)를 할당해 가져온다고 생각하면 이해가 쉬울것이다

### 3.3 this 키워드

object 안에 프로퍼티로 this를 참조하는 함수를 전달하는 방식

```jsx
function classroom(teacher) {
	return function study() {
		console.log(
			`${ tearcher } 선생님이 ${ this.topic }을/를 공부하라고 했습니다.`
		);
	};
}
var assignment = classroom("카일")
var homework = {
	topic: "JS",
	assignment: assignment
}

homework.assignment(); // 카일 선생님이 JS 을/를 공부하라고 했습니다.

// Function.call()을 통해서도 가능하다
var otherHomework = {
	topic: "수학"
}

assignment.call(otherHomework); //카일 선생님이 수학을/를 공부하라고 했습니다.
```
