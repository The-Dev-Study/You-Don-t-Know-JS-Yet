# 2.2 값
프로그램에서 정보의 가장 기본적인 단위는 값이고 값에는 데이터가 저장된다. 값이 있기 때문에 프로그램은 상태를 유지할 수 있다. JS에서 값은 크게 원시 타입과 객체 타입으로 분류된다.

JS에서는 리터럴을 사용해 프로그램에 값을 주입한다.
``` js
greeting("제 이름은 카일입니다.");
```
예시에서 "제 이름은 카일입니다."라는 값은 원시 문자열 리터럴이다.

예시에서는 문자열의 범위를 정하기 위해 큰따옴표를 사용했다. 

문자열 범위를 정할 때는 큰따옴표와 작은따옴표 말고 백틱을 사용할 수도 있다. 다만 백틱은 스타일뿐만 아니라 작동 방식에도 차이가 있다. 예시를 살펴보자

``` js
console.log("제 이름은 ${ firstName }입니다.");
// 제 이름은 ${ firstName }입니다.

console.log('제 이름은 ${ firstName }입니다.');
// 제 이름은 ${ firstName }입니다.

console.log(`제 이름은 ${ firstName }입니다.`);
// 제 이름은 카일입니다.
```
변수 firstName에 '카일'이라는 값이 저장되어 있다는 가정하에 코드를 실행하면 백틱으로 가만 문자열만 문자열 내 변수 표현식인 ${ ... } 부분이 현잿값으로 대체 되면서 제 이름은 카일입니다. 가 출력됩니다. 이런 방식을 보간법이라고 한다.

백틱은 보간 표현식 없이 사용할 수도 있지만 이는 백틱이 만들어진 목적가 부합하지 않으므로 권장하지 않는다.

보간이 필요 없는 경우에는 큰따옴표나 작은따옴표 중 하나를 일관되게 사용하는 게 좋다. 

JS에서는 문자열 말고 불리언이나 숫자 같은 원시 리터럴 값을 사용할 수 있다.
``` js
while(false) {
    console.log(3.141592);
}
```
예시에서는 조건식이 거짓으므로 반복하지 않는다. 

숫자는 주로 프로그램 내에서 횟수를 세거나 수로 표현한 위치 기준으로 정보에 접근할 때 사용한다. 배열이나 객체에 대해서는 조금 후에 다루겠지만, 여기서는 숫자가 어떻게 쓰이는지 알아보기 위해 배열 names의 두 번째 위치에 있는 요소에 접근해보겠다.
``` js
console.log(`제 이름은 ${ name[1] }입니다.`);
// 제 이름은 카일입니다.
```

JS는 문자열, 숫자, 불리언 이외에도 null과 undefined라는 원시타입을 지원한다. 
많은 개발자가 값의 결함이나 부재를 나타낼 때 null과 undefined를 구분 없이 사용하는 경향이 있다.
글자가 짧다고 null이 좀 더 매력적으로 보일 수 있으나, 비어 있는 단일 값을 나타낼 때는 undefined를 사용하는 게 가장 안전한 최선의 방법이다.

> null과 undefined는 자료마다 저자마다 의견이 다르다. 스터디에서 한번 이야기 해보자.

마지막으로 살펴볼 원시 타입 값은 심벌이다, 심벌은 사람이 추측할 수 없게 만든 특수한 숨김 값이다. 심벌은 객체에서 특정한 키를 만들 때 주로 사용한다.
``` js
hitchhikersGuide[ Symbol("삶의 의미") ];
// 42
```

일반적인 JS 프로그램에서 심벌을 직접 사용하진 않는다. 심벌은 대게 라이브러리나 프레임워크 같이 좀 더 저차원의 코드에서 사용한다.
