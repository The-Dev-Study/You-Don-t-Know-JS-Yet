# 2.4 함수
JS로 개발할 때 우리는 함수보다 좀 더 포괄적인 개념인 프로시저를 프로그램에 녹여 내기 위해 심사숙고하며 함수를 작성해야 한다. JS 태동기에는 함수를 다음과 같이 정의할 수 있었다.

``` js
function awesomeFunction(coolThings) {
    return amazingStuff;
}
```

이런 방식을 함수 선언 또는 함수 선언문이라고 하는데, 함수 선언이란 이름은 이 함수가 다른 문의 표현식이 아니라 문 자체이기 때문에 붙게 되었다.

함수 선언 말고도 함수 표현식을 사용해 함수를 정의, 할당할 수 있다. 함수 표현식 예시는 다음과 같다.
``` js
var awesomeFunction = function(coolThings) {
    return amazingStuff;
};
```
예시의 함수는 표현식이므로 변수 awesomeFunction에 할당할 수 있다. 이렇게 함수 표현식으로 선언한 함수가 함수 선언으로 선언한 함수와 다른 점은 함수와 함수 식별자가 코드가 실행되기 전까지는 관계를 맺지 않는다는 점이다.

JS에서 함수는 할당 가능하고 어디든 전달 가능한 값이라는 특징은 매우 중요하다. JS에서 함수는 객체의 한 종류이다. 

함수는 매개변수를 통해 입력을 받는다.
```js
function greeting(myName) {
    console.log(`${ myName }님,  안녕하세요`);
}

greeting('카일'); // 카일님 안녕하세요
```

예시에서 myName은 매개변수이며 함수 내에서 지역 변수 역할을 한다. 함수는 0개부터 원하는 개수만큼 정의할 수 있다.
return 키워드를 사용하면 함수에서 값을 반환할 수 있다.

함수는 오로지 하나의 값만 반환할 수 있다. 
함수는 값이므로 함수를 객체의 프로퍼티로 할당할 수 있다.
``` js
var whatToSay = {
    greeting() {
        console.log('안녕하세요');
    },
    question() {
        console.log('이름이 뭔가요');
    },
    answer() {
        console.log('제 이름은 카일입니다.');
    }
};

whatToSay.greeting();
// 안녕하세요
```
