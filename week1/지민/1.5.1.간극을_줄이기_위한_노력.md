# 1.5.1 간극을 줄이기 위한 노력

JS가 상위 호환성을 보장하지 않기 때문에 오래된 엔진에서 유효한 문법으로 작성한 코드가 돌아가지 않을 가능성이 있음  
ES2019에서 추가된 기능을 사용해 작성한 프로그램을 2016엔진에서 실행하면 프로그래밍이 실행되지 않을 수 있음
-> 이럴 경우 프로그램은 런타임 에러를 발생하며 멈춤

그렇다면 예날 문법만 쓰면서 JS가 발전하는 속도를 따라갈 수 없는 걸까?
-> ** 명세에는 새롭게 추가되었지만 구 엔진과 호환되지 않는 문법은 트랜스파일을 통해 호환성 문제 해결 가능 **

### 트랜스파일

JS 커뮤니티에서 만든 용어로 한 형태에서 다른 형태로 소스 코드를 변환해주는 것을 의미  
이때 산출물은 소스코드  
-> 새로운 JS 문법을 오래된 문법으로 바꿔주며 주로 바벨을 사용

트랜스파일 이전

```
 if(something) {
    let x = 3;
    console.log(x);
 } else {
    let x = 4;
    console.log(x);
 }
```

트랜스 파일 후

```
  var x$0, x$1;
  if(something) {
    x$0 = 3;
    console.log(x$0);
 } else {
    let x$1 = 4;
    console.log(x$1);
 }
```

트랜스파일 전 코드에서 let으로 정의한 변수는 변수의 유효범위가 블록 수준으로 한정 -> 두 변수는 서로 간섭받지 않음  
트랜스파일 후 코드에서는 var로 선언한 변수는 유효범위가 블록 수준이 아님 -> 두 개의 별도의 변수가 생김

** 최신 JS를 사용해 코드를 깔끔하고 의도하는 바를 효과적으로 전달하면서 오래된 JS 환셩에서도 별도의 처리 없이 정상적으로 코드 실행 가능 **
