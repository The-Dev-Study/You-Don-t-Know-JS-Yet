# 1.6.1 웹어셈블리

JS가 주력이 아닌 개발자도 JS 엔진에서 돌아가는 코드를 쉽게 작성할 수 있게 해줌

이름에서도 알 수 있듯 어셈블리 언어와 유사하며 JS 엔진에서 일어나는 일반적인 처리 프로세스와 달리  
파싱, 컴파일 없이 처리됨  
Wasm으로 만든 프로그램의 파싱, 컴파일은 실행 직전 발생, 배포는 JS 엔진의 별도 처리가 별로 필요하지 않은 바이너리 파일 형식으로 진행

Wasm은 성능 향상을 위해 만들어짐
예) Wasm을 통해 고 언어로 작성한 프로그램을 JS 엔진이 해석할 수 있게 한다면 JS 자ㅔ가 스레드를 지원하지 않더라도  
스레드 사용할 수 있게 되는 거 아님?  
-> 다른 언어에서 사용하는 기능, 특히 트랜스파일을 거치는 프로그램 전용으로 설계된 기능을 JS에도 추가해야 한다는 압박이 줄어듦  
-> 다른 언어 생태계에 영향을 받지 않으면서 왜곡되지 않은 시각으로 JS 발전 가능  
-> 동시에 다른 언어들도 웹에서 자유롭게 사용할 수 있는 길을 만들 수 있음

그렇다고 Wasm가 JS를 대체할 수는 없음 단, 웹에서 할 수 있는 것들을 확장하는 역할을 수행할 것  
웹 어셈블리를 JS 회피용으로 사용해선 ㄴㄴ
