# 1.5 하위 호환성과 상위 호환성

하위 호환성은 JS를 지탱하는 기본 원칙 중 하나임

### 하위 호환성

단 한 번이라도 유효한 JS 문법이라고 인정되면 명세서가 변경되더라도 절대 그 유효성이 깨지지 않음  
따라서 1995년에 작성한 코드가 시간이 흘러도 무조건 동작한다는 믿음을 가질 수 있음

하위 호환성을 보장하며 언어를 유지보수하는 것은 쉬운 일이 아님
따라서 예외 존재 -> 개선의 이점에 비해 명세서 변경에 따른 부정적인 부수 효과가 작다고 판단되면 투표를 통해 명세 변경

### 상위 호환성

새로운 명세서에 추가된 문법으로 코드를 작성했을 때 이전 명세서를 준수하는 구형 JS 엔진에서 문제가 발생해선 안됨  
`JS는 상위 호환성을 보장하지 않는다.`

### HTML, CSS

html, css는 반면 상위 호환성을 보장하지만 하위 호환성을 보장하지 않음  
-> 브라우저는 인식이 불가한 html/css는 건너뛰고 인식 가능한 html/css는 명세에 따라 처리하기 때문에 에러 ㄴㄴ
