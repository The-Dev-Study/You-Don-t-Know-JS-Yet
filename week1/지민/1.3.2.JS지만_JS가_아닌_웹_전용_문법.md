# JS지만 JS가 아닌 웹 전용 문법

### alert은 JS 프로그램일까?

```
alert('hello world');
```

alert()은 명세서에는 없지만 모든 웹 환경에서 지원하는 함수
그러나 부록 B에서 커버 ㄴㄴ 왜 그럴까?

브라우저 엔진, node.js등과 같이 JS가 실행되는 환경은 전역 스코프에 API를 추가해 자체적으로 사용할 수 있는 기능 제공
예) alert

-> JS처럼 보이지만 실제로는 웹에서만 지원되는 API
예) fetch(), getCurrentLocation(), getUserMedia()

console 또한 명세서에는 없지만 광범위하게 사용되어 거의 모든 환경에서 이 메서드 지원

즉, alert(), console()은 JS에서 정의된 문법은 아니지만 JS의 함수와 객체 메서드 규칙을 따로고 `JS처럼 보임`

여러 브라우저를 대응하다 보면 브라우저 간 차이 떄문에 JS가 일관성 없어보이는데 사실 대부분의 차이는 JS 동작 방식이 아닌 환경에 따른 작동 방식 떄문에 발생

** 따라서 alert()이라는 코드르 보면 alert 호출은 JS지만 alert 자체는 명세서에 없는 손님이라고 생각해야 함**
