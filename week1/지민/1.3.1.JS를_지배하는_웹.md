# 1.3.1 JS를 지배하는 웹

JS가 브라우저나 서버 심지어 로봇이나 전구에서도 실행되는 등 점차 영역을 확장하고 있지만 가장 큰 비중을 차지하는 분야는 웹임  
-> 따라서 JS 구현체를 만들 때는 웹 브라우저 환경을 가장 중요하게 고려해야 함

그러나 이따금 명세서가 개정되는데 이 개정안이 기존 브라우저 엔진에서 실행되던 JS의 작동 방식과 다른 경우가 발생

### 발생이유?

JS 엔진 제조사들은 긴 세월간 웹 콘텐츠를 다루는 기능을 구현했는데 오류 없이 웹 콘텐츠를 보여주려다보니 엣지 케이스를 다루는 기능도 자체적으로 추가  
이 상황에서 브라우저 제조사들은 명세서 개정안을 반영했을 때 기존에 보이던 콘텐츠가 깨지는 경우 명세서 개정안을 자사 엔진에 반영하지 않겠다는 결정을 내리게 됨

불일치가 발생하면 TC39 위원회는 종종 기존 결정을 철회하고 명세서를 웹에 맞춤
예) 배열 메서드 contains()를 추가하려다 몇몇 프레임워크에서 충돌이 발생 메서드 명을 includes()로 변경

그러나 때때로 브라우저 기반 JS 엔진이 명세서를 준수할 가능성이 거의 없음에도 명세서를 변경하지 않는 경우가 발생
-> 이럴경우 ECMAScript 명세서와 웹에서 돌아가는 JS 차이가 상세하게 기록되어 있는 명세서 부록(B.Additional ECMAScript Features for Web Browsers)을 통해 해결 가능
(부록 B에 적힌 예외들은 브라우저에서만 허용되기 때문에 다른 환경은 명세서를 반드시 준수해야함)

명세서의 B.1, B.2절에서는 웹에서는 쓰이지만 앞서 설명한 역사적 사례를 이유로 명세서에 등재하지 않은 문법과 API를 다룸
예) 0접두사가 붙은 8진 리터럴 빌트인 전역함수 escape(), unescape()
B.3절에서는 웹 엔진과 웹이 아닌 엔진 모두에서 실행되지만, 작동 결과가 달라 문제를 초래할 수 있는 문법과 으미론을 다룸(엄격모드에서 실제 코드가 돌아갈 때 오류를 유발하는 상황도 함께 적혀있음)
