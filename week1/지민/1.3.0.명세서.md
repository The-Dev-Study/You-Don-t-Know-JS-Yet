# 명세서

명세서에 JS 문법과 작동 방식이 정의됨

### TC39 위원회

JS를 관리하는 기술 운영 위원회(technical steering committee)에서 JS의 공식 명세를 관리함

정기 모임에서 명세 변경 안건 투표 -> 합의된 변경 사항 국제 표준화 기구인 ECMA에 제출

- 50~100명으로 구성
- 브라우저를 만드는 조직, 하드웨어 제조사 등 웹 투자를 많이 하는 회사 출신들이 주 구성원
- 위원회 참가자는 모두 자원봉사로 이뤄 짐

제안은 5단계로 이뤄진 절차를 거침 (0~4 단계)  
누군가 새로운 아이디어 제안 -> 위원회의 멤버 중 한 명이 이 아이디어가 가치있다고 생각해 옹호하면 0단계로 올라감
0에서 4단계까지 올라가는 데는 짧게는 몇달 길게는 몇 년이 걸림

모든 제안은 TC39 공개 깃헙에서 확인 가능  
누구나 깃헙을 통해 명세서와 관련된 토론, 제안서 작성과정에 참여 가능하지만 의결은 TC39 회의에서 이뤄지고 멤버만 참여 가능함

### 버전

JS에는 버전이 없음 TC39와 ECMA에 의해 유지되는 공식적인 표준 JS는 하나뿐

브라우저 엔진별로 명세서 개정안을 반영하는 시기가 다르지만 규칙을 어겨선 안 됨  
예를들어 크롬에서 돌아가는 엔진인 v8이 모질라의 엔진 스파이더멍키와 다르거나 호환이 되지 않는 기능을 구현해선 안됨  
덕분에 단 하나의 JS만 배워도 어디서나 동일한 JS 사용 가능
