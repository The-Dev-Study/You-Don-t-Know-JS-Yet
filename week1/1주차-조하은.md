## 1.2 왜 `JavaScript`?

-   **마케팅 목적:** `JS`는 자바 개발자들에게 어필하기 위해 만들어진 언어
    -   실제로 Java와는 완전히 다른 언어지만 표면상 비슷한 문법 채택
-   **C 문법 계열:** C언어와 유사한 문법을 기대하는 개발자를 대상으로 설계

## 1.3 명세서

-   `TC39` : JS를 관리하는 기술 운영 위원회
-   TC39 회원이 아닌 사람의 **아이디어 제안** (0단계 이전) -> 'TC39 회원'이 옹호 (0단계) -> ,,, -> 4단계까지 가야 다음 연도 개정안에 포함될 자격을 가짐
-   JS의 행보는 TC39위원에 의해 좌지우지됨

-   [TC39 공개 깃허브](https://github.com/tc39/propsals)에서 진행 상황 확인 가능

> JS에는 버전이 없고, TC39와 ECMA에 의해 유지되는 공식적인 표준 JS는 단 하나뿐이다.

### 1.3.1 JS를 지배하는 웹

-   개정안과 웹이 불일치하면 → TC39는 기존 결정을 철회하고 **명세를 웹에 맞춤**

### 1.3.2 웹 전용 문법

```js
alert('JS야 안녕!'); // 브라우저 환경에서만 작동
console.log('디버깅'); // JS 명세에 없지만 사실상 표준
```

-   `alert()`, `console.*` 등은 JS 명세가 아닌 브라우저 API이다. 👀❗❗(`console`은 몰랐음)
-   JS 실행 환경(브라우저, Node.js)이 전역 스코프에 추가한 API 기능

> ECMAScript 명세에는 없지만 JS처럼 인식되는 메서드들이 많음. 이 경우 JS 문법 규칙 (함수와 객체 메서드 규칙 등)을 준수해야 한다. 👀❗❗

### 1.3.3 개발자 도구는 JS를 항상 엄격하게 준수하지 않는다.

**개발자 도구의 목적**

-   개발자 편의(DX) 향상
-   빠른 테스트와 디버깅

> -   따라서 순수 JS 작동 방식을 보장하지 않는다.
> -   콘솔에 나타난 결과가 JS 문법을 정확히 지키더라도 JS 표준으로 신뢰하지 말자.
> -   개발자 도구는 JS에 우호적인 **별도의 환경**임을 유의

## 1.4 JS의 다양한 얼굴

**프로그래밍 패러다임이란?**

-   코드를 구조화하는 방법론

**3가지 주요 패러다임**

-   절차적: 코드가 톱다운/선형적으로 구조화 됨 => 코드 단위는 프로시저로 프로시저에 일련의 연산을 작성함
-   객체 지향: 클래스를 기준으로 코드를 구조화 함 => 클래스에는 로직과 데이터를 정의함
-   함수형: 코드를 함수 단위로 구조화함 => 부수효과가 없는 순수함수를 말하며, 함수 자체가 값으로 취급됨

> 자바스크립트는 여러 패러다임을 사용해 코드를 구조화하는 **다중 패러다임 언어**임. 모든 방식을 지원하여 궁극의 유연성을 제공함

## 1.5 하위 호환성과 상위 호환성

> JS는 하위 호환성을 보장한다.

**하위 호환성**

-   한 번 유효한 JS는 영원히 유효함
-   과거에 작성된 코드가 새 브라우저에서도 동작함
-   명세가 변경되어도 기존 문법의 유효성은 깨지지 않음

**상위 호환성**

-   JS는 상위 호환성을 보장하지 않음
-   새 문법은 구형 브라우저에서 작동하지 않음

### 1.5.1 간극을 줄이기 위한 노력: 트랜스파일

JS는 상위 호환성을 보장하지 않으므로, 오래된 엔진에서는 새로운 유효 문법으로 작성한 코드가 돌아가지 않을 가능성이 있다.

**트랜스파일러**

-   구 엔진과 호환되지 않는 새문법을 구 문법으로 변환해줌
-   주로 바벨을 사용한다.

### 1.5.2 간극을 메우기 위한 폴리필

-   구 엔진에서 지원되지 않는 API를 구현해서 추가하는 방법
-   공신력 있는 폴리필 : [ES-Shim](https://github.com/es-shims)
-   트랜스파일러가 자동으로 필요한 폴리필 추가해줌

## 1.7 엄격 모드

**엄격 모드란?**

```js
'use strict'; // 파일 최상단

function myFunc() {
    'use strict'; // 함수 단위
    // 엄격한 규칙 적용
}
```

-   JS 엔진이 코드를 최적화하고 효율적으로 실행할 수 있게 돕는 모드
-   파일 단위/ 함수 단위로 적용이 가능하다.
    -   둘 중 택일해야 함.
    -   함수 단위의 엄격 모드는 점진적으로 엄격 모드를 적용해야 할 때 사용하면 좋다.
-   비엄격 모드에서 작성되었더라도 트랜스파일 처리된 코드는 엄격 모드를 준수한다.
-   ES6 모듈 형식으로 JS코드를 작성하는 경우, ES6 파일은 자동으로 엄격 모드를 준수한다.
